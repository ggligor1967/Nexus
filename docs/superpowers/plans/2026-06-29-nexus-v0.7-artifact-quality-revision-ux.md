# Nexus v0.7 — Artifact Quality + Revision UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship two isolated slices — a deterministic, auditable Markdown export, and a read-only revision history populated on successful plan regeneration.

**Architecture:** Slice 1 turns `planToMarkdown` into a pure renderer that takes persisted metadata and emits YAML frontmatter + numbered, escaped Markdown. Slice 2 records a `revisions` row (server-side, service-role only) whenever a regeneration produces a new validated plan, surfaces that history on the project page, adds a product-facing regenerate path, and renders past snapshots read-only on a dedicated route.

**Tech Stack:** Next.js 16 (App Router, server components), TypeScript (strict), Supabase (Postgres + RLS + service role), Zod, Playwright (E2E + Node-only unit specs).

## Global Constraints

Copied verbatim from the spec — every task implicitly includes these:

- **planToMarkdown must remain pure** — no `new Date()`, no `process.env`, no nondeterministic source inside the renderer or anywhere in the export path.
- **All export metadata comes from persisted values** — `ai_plan_runs.completed_at`, `ai_plan_runs.model_name`, and the validated plan only.
- **Frontmatter values must be safely escaped** via `yamlString` (double-quoted scalars; escape `\` and `"`; flatten newlines).
- **`exportablePlanMarkdown` stays excluded** from the main export output.
- **Revisions are server-written only** — migration `0003` removes client INSERT/UPDATE; SELECT-own is retained.
- **A failed revision insert is logged but non-fatal** — it must never convert a valid 200 generation into a 500.
- **`latestCompletedRun` drives the visible plan/export/revision panel; `latestRun` drives only the failure banner.**
- **The snapshot route returns 404 for a non-owner** (RLS + explicit ownership check).
- **`readOnly` mode suppresses all mutation controls** (no acknowledgement checkbox/buttons, no PATCH).
- **Exactly two implementation commits, with these exact messages:**
  - Slice 1 → `feat: improve deterministic markdown export`
  - Slice 2 → `feat: add read-only revision history`
- **Commit discipline:** Do **not** commit mid-slice. Within a slice, complete and test every task (stage changes, run typecheck/build/relevant E2E), then make the slice's single pinned commit in its final task. This is a deliberate deviation from per-task commits to honor the spec's two-commit constraint.

import type { NexusPlan, RiskLevel } from "@/types/nexus";

export interface ExportMeta {
  exportedAt: string;
  modelName: string;
  riskLevel: RiskLevel;
}

function list(items: string[]): string {
  return items.map((item) => `- ${item}`).join("\n");
}

// Emit a double-quoted YAML scalar; escape backslash and quote, flatten newlines.
function yamlString(value: string): string {
  const escaped = value
    .replace(/\\/g, "\\\\")
    .replace(/"/g, '\\"')
    .replace(/\r?\n/g, " ");
  return `"${escaped}"`;
}

export function planToMarkdown(
  projectName: string,
  plan: NexusPlan,
  meta: ExportMeta

  const heading = projectName.replace(/\r?\n/g, " ");
  const riskUpper = meta.riskLevel.toUpperCase();

  return `---
title: ${yamlString(projectName)}
exported_at: ${yamlString(meta.exportedAt)}
risk_level: ${yamlString(meta.riskLevel)}
model: ${yamlString(meta.modelName)}
format: ${yamlString("nexus-markdown-v1")}
source: ${yamlString("validated_plan_json")}
---

> **Risk Level: ${riskUpper}**
> Review safeguards and red lines before marking this plan build-ready.

## 1. Product Thesis

${plan.productThesis}

${list(plan.deconstruction.userProblems)}

### 2.2 Target Audience

${list(plan.deconstruction.targetAudience)}

### 2.3 Jobs To Be Done

${list(plan.deconstruction.jobsToBeDone)}

### 2.4 Constraints

${list(plan.deconstruction.constraints)}

### 2.5 Assumptions

${list(plan.deconstruction.assumptions)}

### 2.6 Unknowns

${list(plan.deconstruction.unknowns)}

### 2.7 Success Metrics

${list(plan.deconstruction.successMetrics)}

${list(plan.strategy.mvpBoundary)}

${list(plan.strategy.nonGoals)}

## 4. Prototype Options

${plan.prototypeOptions

export function sanitizeFilename(title: string): string {
  const safe = title
    .toLowerCase()
    .replace(/[^a-z0-9-_]+/gi, "-")
    .replace(/^-+|-+$/g, "")
    .slice(0, 80);

  return safe || "nexus-plan";

- [ ] **Step 1: Write the migration**

Create `supabase/migrations/0003_restrict_revision_writes.sql`:

```sql
-- v0.7: revision rows are server-written audit events.
-- Authenticated users may read revisions for their own projects, but direct
-- client INSERT/UPDATE is denied; writes go through service-role server code.
-- Mirrors SEC-003 (migration 0002) for ai_plan_runs.
DROP POLICY IF EXISTS "Users can insert revisions for own projects" ON public.revisions;
DROP POLICY IF EXISTS "Users can update revisions for own projects" ON public.revisions;
-- Note: migration 0001 never created an UPDATE policy on revisions, so the
-- second DROP is a harmless no-op kept for symmetry. SELECT-own is intentionally retained.
```

import Link from "next/link";
import { notFound, redirect } from "next/navigation";
import { createServerSupabaseClient } from "@/lib/supabase/server";
import GeneratedPlanView from "@/components/GeneratedPlanView";
import type { NexusPlan } from "@/types/nexus";

export const dynamic = "force-dynamic";

export default async function RevisionSnapshotPage({
  params
}: {
  params: Promise<{ id: string; revisionId: string }>;
}) {
  const { id, revisionId } = await params;
  const supabase = await createServerSupabaseClient();

  const {
    data: { user }
  } = await supabase.auth.getUser();

  if (!user) {
    redirect(`/login?next=/projects/${id}/revisions/${revisionId}`);

          />
        </section>

          <h2>New version</h2>
          <GeneratedPlanView

            buildReadyAcknowledged={false}
            readOnly
          />
        </section>
      ) : null}

  );
}
